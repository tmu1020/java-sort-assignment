# ソート実装課題

Wikipediaの「ソート」の一覧より、「クイックソート」と「マージソート」を選択し、Javaを用いて実装した。

## 各ソートを実装して良かった点

### 1. クイックソートの良かった点
配列のメモリ領域の内部だけで要素の入れ替えを行うため、余分なメモリをほとんど消費しない点が非常に優れていると感じた。実務で基本データ型のソートに広く採用されている理由を、実装を通して身をもって体感できた。

### 2. マージソートの良かった点
実際にコードを書いてみて、データを安全に並び替えるために「一時配列（temp）」を用意し、3つのポインタ（i, j, k）をコントロールしながら元の配列に書き戻すロジックの難しさを体感できた。クイックソートに比べてコード量が多く、メモリも余分に消費する一方で、この丁寧な手順を踏むからこそ「データの順序を崩さない（安定ソート）」という仕組みが成り立っているのだと、実装を通して納得できた。

---

## 実行方法

本プログラムは、コマンドライン引数から任意の整数配列を受け取ってソートを行うことができる。引数がない場合は、デフォルトのテストデータで実行される。

```bash
# コンパイル
javac SortAssignment.java

# 引数なしで実行（デフォルトデータ）
java SortAssignment

# 引数を指定して実行
java SortAssignment 5 2 9 1 7
```
